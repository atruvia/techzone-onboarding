# TechZone BMAD Onboarding

A statically-served website that onboards Atruvia developers who are new to **BMAD** (the BMAD-METHOD agentic agile development framework). It routes each developer to the BMAD workflow that fits their situation and lets them go deeper on any topic via the AI tutor.

**BMAD**:
The **BMAD-METHOD** — a public AI-driven agile development framework (`github.com/bmad-code-org/BMAD-METHOD`, docs at `docs.bmad-method.org`) where specialised AI agent personas (Analyst, PM, Architect, Scrum Master, Dev, QA, UX, …) guide work through a structured lifecycle: brainstorm/analysis → planning (PRD) → architecture → implementation → QA → deploy/retro. The public project is the **single source of truth**; all Page content cites it. Not an Atruvia-internal flavour.
_Avoid_: BMad, the method (use "BMAD")

## Language

**Onboarding Site**:
The whole deliverable — a static website, served by GitHub Pages, that helps Atruvia developers new to BMAD get started.
_Avoid_: app, portal

**Mission**:
The reason the site exists: an Atruvia developer new to BMAD lands, recognises their situation, and confidently starts the matching BMAD workflow — and can go deeper on any topic via the AI tutor.

**Hub**:
The home landing page (the site index). A single branded page built with `/atruvia-design-system` whose job is to route the visitor to the four Pages via a 2×2 grid of four clickable cards.
_Avoid_: home page (use "Hub" for the concept, "home" only for the folder name)

**Page**:
A self-contained mini-site under `./pages/<name>/`, generated by one `/atruvia-teach` session and committed as frozen static files (an `index.html` topic landing plus a handful of lesson/reference HTMLs). One Page teaches one BMAD scenario.
_Avoid_: lesson (a lesson is one HTML file *inside* a Page), section, tab

**Root index**:
The Hub's `index.html`, placed directly at the repository root so GitHub Pages serves it as the site root. There is no `home/` folder — "home" is the concept the Root index realises.
_Avoid_: home folder, home page file

**Scenario**:
What one Page teaches — the visitor's situation, picked from the Hub under the framing »Wo stehst du gerade?« (Where are you right now?). There are exactly four, three by work-scale plus the tutor:

- **Innovate** (slug `innovate`, card »Erkunden & Recherche«): a fuzzy idea — brainstorming/research, the front of the BMAD method.
- **Small Task** (slug `small-task`, card »Kleine Aufgabe«): one small concrete change — a short BMAD dev cycle.
- **Small Project** (slug `small-project`, card »Kleines Projekt«): a whole project — full BMAD planning + dev cycle + retro.
- **Learning** (slug `learning`, card »Lernen mit dem KI-Tutor«): learn any topic deeply via the `/atruvia-teach` AI tutor (this Page also teaches *how to use that skill*).

_Avoid_: track, module, course, tab

**Back-link**:
The consistent »Zurück zur Übersicht« navigation element on every Page header that returns the visitor to the Root index (Hub). The contract that makes Hub and Pages feel like one site.

**Card**:
One of the four clickable tiles in the Hub's 2×2 grid. Each Card = a Lucide icon + German title + one-line description, the whole tile linking to its Page (`pages/<slug>/`). One Card per Scenario.
_Avoid_: button, tile, link

**Lesson**:
A single self-contained teaching HTML file *inside* a Page (`pages/<name>/lessons/0001-*.html`), each delivering one tangible win. A Page has ~2–3.
_Avoid_: page, chapter

**Reference**:
A compressed quick-reference HTML *inside* a Page (`pages/<name>/reference/*.html`) — a cheat sheet or glossary the Lessons link to and the visitor returns to. A Page has ~1.
_Avoid_: doc, appendix

## Conventions

**Language**: All user-facing content is **German only** (Atruvia is a German-speaking company): plural »Wir«, informal »du«, Gender-Stern (*Entwickler\*innen*), »German guillemets«. The only exceptions are inherently-English items — BMAD's own command/role names (`PM`, `Architect`, `Scrum Master`, PRD, greenfield/brownfield) and code keywords — which stay in English.

**Hosting**: Repo `github.com/atruvia/techzone-onboarding`, served by GitHub Pages as a project site at `https://atruvia.github.io/techzone-onboarding/`. Consequence: **all links and asset references must be relative** (e.g. `pages/innovate/`, `../../index.html`) — never root-absolute (`/pages/...`), which 404s under the project subpath.

**Self-containment**: Each Page is a **literally self-contained folder** — it bundles its own copy of the design system (CSS + Atruvia fonts) under `./pages/<name>/assets/`. No shared root `/assets/`. Accepts ~4× duplication in exchange for folders that work standalone if extracted. Only the Hub keeps its own (website-kit) assets at the root level. Each Page also keeps its full `/atruvia-teach` scaffolding (`MISSION.md`, `RESOURCES.md`, `learning-records/`, `NOTES.md`) so it stays a living workspace that can be re-opened and continued.

**Audience environment**: Developers already have **BMAD, the Claude CLI, Python and Node.js pre-installed** — a fully dev-ready machine. Consequences: Pages teach *usage*, never installation/setup; a developer can re-open any Page folder in their Claude CLI to continue its teach session live; and the site can be previewed locally with `python3 -m http.server` before pushing.
